# 🎵 音乐播放器需求文档（v1.0.0）

> **项目名称**：HarmonyPlayer
> **目标用户**：本小组成员（3–5人）
> **使用场景**：本地/在线音乐播放、个人歌单管理、轻量级音乐探索
> **部署方式**：初期以 Web 应用为主，后续可选封装为桌面应用（如 Electron / Tauri）  

------

## 一、核心目标

开发一个**简洁、稳定、自用**的音乐播放器，支持：

- 播放本地音乐文件（如 MP3、FLAC）
- 播放在线公开音频资源（如公开链接、YouTube 音频代理等，*不涉及版权内容*）
- 创建与管理个人歌单
- 提供基础播放控制（播放/暂停、音量、进度、循环/随机等）

> ⚠️ 注意：**不提供音乐下载、不集成第三方付费 API（如 Spotify/网易云）、不处理 DRM 内容**，仅内部使用音频资源。

------

## 二、功能需求



### ==前期==

* 音乐库：
  支持从本地上传音频文件（拖拽或选择文件）
  支持编辑音乐信息(标题、歌手、专辑、封面)
* 支持歌单管理：创建、重命名、删除歌单、添加/移除歌曲、 点击歌单即可播放其中所有歌曲
* 支持播放器：
  播放 / 暂停 / 上一首 / 下一首
  进度条拖拽 + 实时显示当前时间 / 总时长
  音量调节（含静音按钮）
  播放模式：顺序播放 / 单曲循环 / 列表循环 / 随机播放
  最小化播放栏（固定在页面底部，切换页面不中断）



### ==中期==



* 音乐库：
  自动解析音频元数据（ID3 标签）：标题、艺术家、专辑、封面
* 用户体验：
  深色/浅色主题切换
   播放时显示专辑封面（若有）
  用户登录认证  个人中心
* 播放器：
  基础歌词同步显示（可能需手动上传 `.lrc` 文件）
  键盘快捷键支持（空格=播放/暂停，→=快进5秒等）



==后期==

* 每日(个性化)推荐
* 视频转音频
* 个性化设置





## 三、非功能需求

| 类别         | 要求                                                       |
| ------------ | ---------------------------------------------------------- |
| **性能**     | 音频加载延迟 < 1s，界面操作流畅（60fps）                   |
| **安全性**   | 不上传用户音频到服务器；前端本地处理所有文件               |
| **离线能力** | 支持离线播放已加载的本地音乐（通过浏览器缓存或 IndexedDB） |
| **兼容性**   | 主流现代浏览器（Chrome, Edge, Firefox, Safari）            |
| **代码维护** | 使用模块化架构，便于小组协作开发                           |
