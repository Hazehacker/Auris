







## 四、技术选型建议（初版 Web 版）

| 模块      | 推荐技术                                                |
| --------- | ------------------------------------------------------- |
| 前端框架  | React + Vite（轻量、快速）                              |
| UI 组件库 | Tailwind CSS + Headless UI（或 Ant Design）             |
| 音频播放  | [Howler.js](https://howlerjs.com/) 或原生 Web Audio API |
| 本地存储  | localStorage / IndexedDB（存歌单、设置）                |
| 文件处理  | FileReader API（解析本地音频）                          |
| 构建部署  | Vercel / Netlify（免费托管）或本地局域网服务            |

> 💡 若后期转桌面端：可用 **Tauri**（Rust + Web 前端，体积小）或 **Electron**（直接复用 Web 代码）

------

## 五、开发里程碑（建议）

| 阶段            | 目标                                            | 时间（参考） |
| --------------- | ----------------------------------------------- | ------------ |
| Phase 1         | 实现本地音乐上传 + 基础播放器（播放/暂停/进度） | 1周          |
| Phase 2         | 歌单管理 + 元数据显示 + 主题切换                | 1周          |
| Phase 3         | 优化 UI/UX + 快捷键 + 响应式布局                | 3天          |
| Phase 4（可选） | 封装为桌面应用 / 添加歌词 / URL 播放            | 视兴趣而定   |

------

## 六、备注

- 所有音频数据**仅存储在用户本地浏览器**，不涉及后端服务器（除非你们想加局域网共享功能）。
- 项目代码可开源在 GitHub 私有仓库，便于版本控制与协作。
- 鼓励使用 TypeScript 提升代码健壮性。
